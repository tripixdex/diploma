# MQTT Interface

## Роль в проекте

Базовый транспорт обмена между:

- роботом;
- сервером;
- операторским интерфейсом.

## Почему выбран

- простой;
- легковесный;
- хорошо подходит под `publish/subscribe`;
- удобен для телеметрии и команд;
- быстрее и реалистичнее для прототипа, чем тяжелая робототехническая шина.

## Что должно быть формализовано

- список топиков;
- форматы сообщений;
- QoS для критичных команд;
- ACK/обработка ошибок;
- поведение при потере связи.

## Где это должно появиться в ВКР

- концептуальное проектирование;
- техническое задание;
- структурное проектирование;
- рабочее проектирование;
- апробирование.
