I don't think a repair will break the software, but you could try a full reinstall. 

First, copy the current Image32 directory to Image32backup in case we need it. 

Then put the CD in and choose Uninstall. 

Delete the Image32 directory. 

Eject the CD and put it in again, you should now be able to install the software. Set the install directory to C:\Image32 and leave all the other options at their default settings. 

Reboot the PC and see if it works. 

If this does not work you could try reinstalling the camera driver. 

Make sure your camera is plugged in. 

Open the device manager by right clicking on "My Computer" and choosing properties, then go to the Hardware tab and click on the "Device Manager" button. 

From the list of available device types, select Imaging devices. 

If you are installing this driver over top of a previous version, your digital camera will be listed as "CMU 1394 Digital Camera Device" otherwise, it will typically be listed as "Sony 1394 CCM-DS250 Desktop Camera". or as "Generic 1394 Digital Camera #n" under winXP. Right-click it and select Properties. 

In the properties panel, select the Driver tab, and then click Update Driver. 

Follow the on screen instructions, do NOT use the search option. The driver is not signed and windows will ignore it in favor of the built-in (and very limited) driver for these cameras. Use the "Display a list of..." option instead. 

Click on the "Have Disk..." button, and point the directory to where 1394Camera.inf is stored (Drivers/1394camera on the CD). 

Ignore the warnings about the driver not being digitally signed. When installation is finished, verify in the driver tab that the provider is "Carnegie Mellon University" and that the version is "6.0.0.0". Also verify that your camera is now listed in the device manager as "CMU 1394 Digital Camera Device" (for some reason, this doesn't happen under winXP...). 

Restart Your Computer.
