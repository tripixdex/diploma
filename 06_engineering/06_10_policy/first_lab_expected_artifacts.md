# First Lab Expected Artifacts

## Required Return Package
- Board overview photos.
- Board markings / revision photos.
- Connector photo set:
  - power,
  - headers,
  - motor/driver-related connectors if visible,
  - sensor-related connectors if visible,
  - ESTOP-related connectors or cabinet interfaces if visible.
- Mounting / enclosure context photos.
- Filled `first_lab_data_capture_sheet.md`.
- Completed board-fact checklist results.
- Motor driver / actuator interface notes.
- Sensor interface notes.
- Signal mapping notes with unknowns clearly marked.
- Structured unknowns list.
- Procedure-boundary notes:
  - what was allowed,
  - what was forbidden,
  - who approved it.

## Strongly Recommended Artifact Naming
- `YYYYMMDD_lab_board_overview_01.jpg`
- `YYYYMMDD_lab_board_markings_01.jpg`
- `YYYYMMDD_lab_connector_power_01.jpg`
- `YYYYMMDD_lab_connector_header_01.jpg`
- `YYYYMMDD_lab_connector_motor_01.jpg`
- `YYYYMMDD_lab_connector_sensor_01.jpg`
- `YYYYMMDD_lab_estop_chain_01.jpg`
- `YYYYMMDD_lab_capture_sheet.md`
- `YYYYMMDD_lab_unknowns.md`

## What These Artifacts Must Prove
- Which board is actually present.
- How the board is physically situated.
- Which connectors and interfaces are visible.
- What is known about power and ESTOP boundaries.
- Which fields remain unknown after the visit.

## What These Artifacts Must Not Be Used To Claim
- Hardware readiness.
- GPIO readiness.
- Motion-control readiness.
- Proven safety behavior on the real AGV.
- Raspberry Pi binding completion.
- Orange Pi portability proof.
