# Board Fact Checklist

This checklist exists to prevent board-binding claims from vague observation.

## Board Identity
- Exact board vendor is recorded.
- Exact model name is recorded.
- Exact revision or board markings are photographed if present.
- It is explicit whether the observed board is Raspberry Pi 4, something else, or still uncertain.
- It is explicit whether Orange Pi is actually present in the lab setup or only a future portability idea.

## Physical Context
- Board overview photo is taken.
- Board mounting / enclosure context is photographed.
- Current connected peripherals are photographed.
- It is recorded how the board is currently mounted and used in the lab setup.

## Power Facts
- Power entry connector is identified or marked unknown.
- Power source path is observed or marked unknown.
- Any visible labels, supply markings, or restrictions are captured.
- Any explicit lab warning about safe power handling is recorded.

## GPIO / Header Facts
- GPIO or other headers are photographed.
- Header accessibility is recorded.
- Any existing wiring on headers is photographed.
- Unknown pin usage remains marked unknown.

## Attached Interface Hardware
- Any HAT, controller board, USB interface, serial adapter, or other daughterboard is photographed.
- Connector relationships between the board and external hardware are noted.
- Nothing is assumed to be a motor or sensor interface without evidence.

## Runtime Context
- Storage medium facts are captured if visible.
- Any visible OS / console / service context is recorded only as observed, not assumed from memory.
- Existing software stack on the board is marked unknown unless directly shown.

## Minimum Evidence Required
- Readable board markings photo.
- Overview photo.
- Connector / header photo set.
- Power-entry photo or explicit note that it could not be observed.
- One completed data-capture sheet entry for board facts.
